# 贪吃蛇游戏需求规格书 (Snake Game Spec)

## 技术栈
- 纯单个文件 `index.html`，内联 CSS 和 JS，方便部署。
- 不使用任何外部框架或库。

## 视觉与交互
- 经典像素风或现代极简风。
- 居中的 400x400 大小的游戏画布 (Canvas)。
- 页面必须包含：
  - 醒目标题 "Snake Claw"
  - 实时更新的当前分数展示
  - 最高分展示（使用 localStorage 存储，刷新页面不丢失）
- 背景颜色清晰护眼。
- 当游戏结束时（Game Over），屏幕应显示半透明遮罩叠加层，包含居中的闪烁文字提示 "Game Over" 以及一个巨大的 "重新开始" 按钮。

## 游戏机制
- 初始蛇身长度为 3 个单位，移动速度适中（例如 100ms 每次刷新）。
- 蛇撞到画布边缘墙壁，或者蛇头撞到自身任意部位，即触发 Game Over。
- Canvas 随机生成代表目标（如苹果）的红色方块。
- 蛇吃到苹果后：
  - 身体长度 +1
  - 计分板分数 +10
- 按键盘上下左右 (Arrow Up/Down/Left/Right) 控制移动方向。
- 必须防止蛇直接 180 度反向掉头（例如：当前向右移动时，按下左方向键无效）。

## 交付要求
请阅读并理解此文档后，直接在当前目录生成完整的 `index.html`。
